# Contributing

Thank you for your interest in 4ge.

## Contributions are currently closed

4ge is **source-available** under the [Functional Source License (FSL-1.1-MIT)](./LICENSE),
not open source. At this time we are **not accepting external code
contributions** (pull requests).

This is a deliberate posture, not an oversight. The FSL preserves the ability to
relicense the project (it converts to MIT on the Change Date), and accepting
outside contributions without a contributor agreement would complicate that
path. Keeping contributions closed for now keeps the licensing story clean.

### Pull requests

Unsolicited pull requests will generally be closed without merge. If you have
already done work you would like us to consider, please reach out first (see
below) rather than opening a PR.

### If contributions open later

If and when we open contributions, it will be **CLA-gated**: contributors will
be asked to sign a Contributor License Agreement before any external code is
merged. We will update this document when that happens.

## What you can do today

- **Report bugs.** Open a GitHub issue with a clear repro, the plugin version,
  and your environment. Bug reports are welcome.
- **Report security issues privately.** See [SECURITY.md](./SECURITY.md) —
  please do not file security problems as public issues.
- **Request features.** Open an issue describing the use case. We read these
  even when we cannot act on them immediately.
- **Ask questions.** General questions can go to **info@3sixtyco.dev**.

## Using the plugin in your own work

The FSL permits you to use, modify, and share the Software for non-competing
purposes (see [LICENSE](./LICENSE) for the exact terms). Files installed by the
Blueprint installer into your own repositories are MIT-licensed immediately —
those are yours to do with as you wish.
